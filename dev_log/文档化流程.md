# 代码改动文档化流程

## 何时写
完成功能添加/重要重构后立即记录。

## 写什么
```markdown
# 功能名 实现
日期 | 类型 | 影响范围

## 概述
改动目的 + 核心功能点

## 规则
关键逻辑/默认值

## 文件修改
文件路径 + 代码片段（用 // ... 省略非关键代码）

## 注意事项
优先级/兼容性/陷阱
```

## 原则
- **必须**：改了什么文件、具体代码、核心逻辑
- **不写**：冗长背景、使用教程、测试用例
- **控制**：50-150行，面向3个月后的自己

## 价值
快速回顾改动 | 问题追溯 | 知识沉淀

## 存放位置
`dev_log/功能名-implementation.md`
